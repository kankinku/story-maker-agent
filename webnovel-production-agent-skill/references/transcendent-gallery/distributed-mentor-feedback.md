# Distributed Mentor Feedback

Use this reference for community-assisted tower climb stories where experienced transcendents act as plural, imperfect mentors.

## Core Rule

The gallery must not be an answer vending machine. It is a mentor network, reaction audience, analysis room, emotional chorus, and mystery channel. Advice should be plural, biased by each mentor's background, and incomplete under the current stage conditions.

## Loop

```text
new stage and rule encounter
-> limited observation and danger estimate
-> photo, video, or situation shared to gallery
-> multiple transcendents offer different interpretations
-> protagonist compares trust, premise, and current condition
-> protagonist builds a personal plan
-> real combat or trial begins
-> variable violates part of the advice
-> improvisation and technical breakthrough
-> skill, relationship, information, or reputation gain
-> result uploaded
-> post-review sets next growth direction
```

## Rule Card

```yaml
id: distributed-mentor-feedback
purpose: Turn conflicting mentor advice into growth material.
procedure:
  - Present the current problem with observable evidence.
  - Have at least two mentors interpret it from different specialties.
  - Expose each advice's premise and limitation.
  - Make the protagonist select only the usable part for their body, weapon, skill, and stage condition.
  - Add a field variable that prevents perfect execution.
  - Convert the result into a later technique, habit, or judgment rule.
avoid:
  - One mentor monopolizes the correct answer every time.
  - The protagonist perfectly learns a skill immediately after reading comments.
  - The protagonist follows orders without judgment.
verify:
  - Could two mentors reasonably disagree?
  - Did the protagonist choose, combine, or reject advice?
  - Did the field result create a new lesson rather than only praise?
```

## Mentor Types

| Type | Function |
|---|---|
| sword or martial mentor | posture, basics, imagery, combat judgment |
| magic mentor | mana operation, phenomenon analysis, technique combination |
| system veteran | stage type, reward, difficulty rule |
| survival explorer | traps, environment, enemy behavior warning |
| lore mentor | race, faction, ancient incident |
| cynical comic mentor | tension release and comment rhythm |

Mentor voices need distinct expertise and limits. Repeated catchphrases are not enough.

## Anti-Patterns

- Gallery predicts the exact solution before every fight.
- Mentor voices differ only by slang.
- The protagonist becomes a command receiver.
- Comment sections are mostly empty admiration.
