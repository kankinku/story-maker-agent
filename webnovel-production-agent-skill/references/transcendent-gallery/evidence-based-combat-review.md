# Evidence Based Combat Review

Use this reference when combat footage, screenshots, or uploaded proof becomes a second reading of an already seen event.

## Function

Video feedback should add new information. The reader should not reread the same fight in comment form.

The upload/review cycle:

```text
combat happens
-> footage is recorded
-> fight ends
-> gallery upload
-> mentors isolate key moments
-> protagonist learns a hidden problem or achievement
```

## Valid New Information

Each review section should provide at least one:

- mistake the protagonist missed
- real cause of a successful technique
- structure behind what looked accidental
- reproducible condition for next time
- achievement the protagonist underestimated
- hidden danger not yet exposed
- mismatch between mentor assumption and stage reality

## Review Shape

```text
normal analysis comment
-> exaggerated reaction
-> recurring named-user bit
-> professional rebuttal
-> protagonist summary or choice
```

The last useful beat should leave a decision, training target, warning, or question.

## Anti-Patterns

- "That was cool" repeated in multiple voices.
- Comments describe the exact same visible action with no interpretation.
- Review is only a praise scene for reputation.
- Every review ends with the same mentor being correct.
