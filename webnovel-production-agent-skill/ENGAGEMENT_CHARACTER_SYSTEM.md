# Engagement And Character System

이 문서는 캐릭터 중심 기획 원칙과 추상적인 "재미/도파민"을 관찰 가능한 서사 계약으로 바꾼다. 단일 프롬프트에 모든 규칙을 쌓지 않고 `Canon -> Serial Plan -> Scene Contract -> Draft -> Audit` 단계에 필요한 정보만 전달한다.

## 1. 우선순위

1. 독자가 다시 보고 싶은 캐릭터를 만든다.
2. 그 캐릭터가 욕망과 판단으로 사건을 시작하거나 방향을 바꾼다.
3. 현재 목표와 저항을 명확히 하고, 장면마다 선택의 결과를 남긴다.
4. 기다린 갈등을 주인공의 행동으로 해소해 보상을 제공한다.
5. 관계와 캐릭터에서 나온 유머·긴장 완화로 장면 자체의 재독 가치를 만든다.
6. 이번 화의 만족과 다음 화의 구체적 질문을 동시에 남긴다.

문장 미학과 설정의 양은 이 인과사슬을 보조한다. 가독성은 별도 전달 게이트이며, 짧은 문장만을 뜻하지 않는다.

## 2. 캐릭터 우선 정본

시놉시스 전에 `engagement_scope: core`인 핵심 캐릭터마다 다음을 최소 정본으로 고정한다. 아직 기능이 정해지지 않은 단역은 `supporting_deferred`로 두고 장면 투입 전에 승격한다.

- `desire`: 지금 스스로 얻으려는 것
- `stakes`: 실패하면 잃는 것
- `principle`: 손해를 감수해도 지키려는 행동 원칙
- `contradiction_or_flaw`: 매력과 문제를 함께 만드는 모순 또는 결점
- `choice_signature`: 압박받을 때 반복해서 드러나는 판단 방식
- `speech_signature`: 다른 인물과 구분되는 말의 목적·어휘·거리감
- `relationship_variation`: 상대에 따라 달라지는 태도
- `repeatable_appeal`: 독자가 다음 등장을 기다릴 반복 보상
- `change_direction`: 결말까지 무엇이 바뀌거나 끝까지 지켜지는가

배경 노트는 행동과 대사의 근거가 되는 만큼만 작성한다. 모든 캐릭터에게 1~2페이지를 강제하지 않으며, 현재 20화에서 사용되지 않는 전사는 `deferred`로 둔다.

## 3. 로그라인과 시놉시스

로그라인은 `주인공 + 현재 결핍/상황 + 스스로 택한 목표 + 고유한 수단/제약 + 결말 방향`을 포함한다. "결말 포함"은 결말의 문장 전체를 확정하라는 뜻이 아니라 주인공 변화와 핵심 갈등의 해소 방향을 잠그라는 뜻이다.

시놉시스는 집필을 지연시키는 백과사전이 아니라 의사결정 도구다. 다음 항목이 있으면 초고 준비로 이동할 수 있다.

- 핵심 캐릭터와 관계 축
- 작품 약속과 반복 보상
- 시작 상태, 3개의 앵커 사건, 결말 방향
- 20화 사건표와 각 화의 상태 변화/보상/다음 이유
- 정본 변경 정책

기승전결마다 장면 수를 획일적으로 고정하지 않는다. 계획이 승인된 뒤에는 초고를 완주하고, 수정은 후속 감사에서 처리한다.

## 4. 익숙한 세계와 정보 공개

- 세계관은 캐릭터의 선택을 가능하게 하거나 어렵게 하는 규칙부터 만든다.
- 새 개념은 익숙한 장르 문법이나 생활 언어로 번역하되, 독창성을 없애지 않는다.
- 빙의·환생·트립은 독자 안내와 주인공 목표에 실제 기능이 있을 때만 사용한다. 낯선 설정의 자동 보정 수단으로 강제하지 않는다.
- 설정은 사용, 위반 결과, 갈등, 선택을 통해 공개한다.
- 새 정보는 현재 장면의 판단을 바꾸거나 곧바로 활용되어야 한다. 그렇지 않으면 뒤로 미룬다.

## 5. 장면 진행 계약

모든 장면은 `plot`, `character`, `relationship`, `world_rule_in_use` 중 하나 이상을 전진시켜야 한다. 단순 분위기 장면도 인물의 선택·관계·압력 중 하나를 바꿔야 한다.

장면은 다음 순서로 검사한다.

1. 현재 인물의 목표와 저항이 보이는가?
2. 인물이 판단하고 선택하는가?
3. 선택이 결과나 비용을 만드는가?
4. 캐릭터 고유성 또는 관계 차이가 드러나는가?
5. 새 정보가 선택에 사용되는가?
6. 장면 종료 시 상태가 달라졌는가?

기능을 마친 이동·준비·반복 설명은 압축한다. 반대로 감정 이해, 갈등 축적, 관계 변화, 보상 순간은 결과가 체감될 만큼 시간을 준다.

## 6. 재미를 평가하는 10개 차원

각 점수는 1~5이며 반드시 장면 근거를 남긴다.

| 차원 | 4점 이상을 위한 관찰 근거 |
|---|---|
| `character_appeal` | 말·행동·반응이 구분되고 다음 선택 또는 재등장을 기다리게 한다. |
| `protagonist_agency` | 주인공이 목표·방법을 선택하며 그 선택이 다음 사건을 만든다. |
| `earned_payoff` | 기다린 갈등과 대상이 명확하고, 축적에 비례한 결과를 주인공이 만든다. |
| `humor_relief` | 유머나 환기가 캐릭터·관계에서 나오며 핵심 감정을 훼손하지 않는다. 장르상 불필요하면 의도된 대체 완급 기능을 기록한다. |
| `goal_clarity` | 독자가 현재 목표, 문제, 장면의 필요성을 즉시 이해한다. |
| `pacing_control` | 끝난 기능은 압축하고 감정·갈등·보상에 필요한 시간은 확보한다. |
| `exposition_naturalness` | 설정이 행동·위반 결과·갈등에 붙고 즉시 선택에 쓰인다. |
| `episode_completion` | 질문 하나 이상이 해결되고 구체적 상태 변화와 읽은 보상이 남는다. |
| `relationship_chemistry` | 함께 있을 때 개성이 증폭되고 주도권·신뢰·거리 중 하나가 변한다. |
| `next_episode_pull` | 이번 화 결과가 다음의 구체적 질문·결정·기회·위기를 발생시킨다. |

`readability`는 전달 게이트로 별도 검사한다. 화자·행동 주체, 시간과 장면 전환, 고유명사 부담, 반복 설명, 모바일 문단, 재독 필요성을 확인한다.

## 7. 화 단위 만족과 열린 고리

한 화는 다음 네 요소를 갖는다.

1. 해결하거나 확인할 작은 문제
2. 주인공 또는 핵심 캐릭터의 행동과 선택
3. 해결·실패·반전 중 하나의 결과와 구체적 상태 변화
4. 일부 만족 이후 그 결과가 만든 다음 화의 구체적 질문

문장을 중간에서 끊거나 매번 새 위기만 던지는 것은 열린 고리가 아니다. `resolved_question`, `concrete_state_change`, `reader_reward`, `next_episode_specific_question`이 서로 인과로 이어져야 한다.

## 8. 초반 구조의 사용법

초반 수치는 기본 진단 창이지 장르 불문 공식이 아니다.

- 1~3화: 콘셉트, 주인공 욕망·선택, 작품 약속을 실제 장면으로 각인
- 5화 안팎: 고유 수단이나 성장 엔진이 어떤 이득을 만드는지 증명
- 10화 안팎: 첫 갈등 묶음의 해소와 보상
- 20~25화: 반복 보상과 더 큰 장기 이유를 증명

의도적으로 늦출 때는 대체 보상, 지연 이유, 증명 예정 회차를 기록한다.

## 9. 초고와 교정 분리

초고는 캐릭터 행동, 대사, 인과, 사건 완주를 우선한다. 문장은 직관적으로 쓰되 유아화하지 않는다. "관찰자처럼 나열"하는 방식이 캐릭터 내면과 선택을 없애면 사용하지 않는다.

교정 단계에서는 다음을 처리한다.

- 과장된 비유와 인위적 미사여구
- 같은 종결·단어·문장 구조의 연속 반복
- 감정 이름의 반복 설명
- 화자 불명확, 장면 전환 혼란, 긴 모바일 문단
- 캐릭터별 말투와 상대별 거리감의 붕괴

최종 한국어 원고에는 별도의 인간화와 AI-tell guard가 그대로 적용된다.

## 10. 실패 우선순위

감사에서는 첫 번째 고심각도 원인만 고친다.

1. 캐릭터 욕망·주도성 부재
2. 현재 목표와 장면 필요성 불명확
3. 선택의 결과·화 단위 상태 변화 부재
4. 축적 없는 보상 또는 보상 없는 지연
5. 관계·유머·정보 공개 기능 약화
6. 다음 화 질문의 비구체성
7. 가독성과 문장 표면 문제

문장만 다듬어 구조 결함을 숨기지 않는다.
